Signals and Systems

Oscillator Playback Filters Voice

This app aims to provide users with some tools to perform simple experiements to supplement their learning for EE2010 Signals and Systems.

The experiements in this app are arranged specifically in such a way to demonstrate, progressively, effects of signals and systems. The sequence is as follow:

  1. Oscillator is essentially a function generator to showcase the generation of the basic blocks of sound. Also a handy tool to check how a function sounds like.
  2. Playback allows users to control the playback rate of an audio sample to showcase a speed control system and the physical effects of manipulating frequency.
  3. Filters take systems one step further to showcase different kinds of input signal manipulation.
  4. Voice allows users to use their own voices to be passed through a predefined system and observe effects in realtime.

This app is not meant to replace any part of EE2010.

Message from author

I believe EE2010 is one of the more important modules for EEE students as the concepts introduced in this module is deployed in most advanced topics, such as modelling, control, network, signal processing, machine learning, computer vision and more.

The overarching idea may be simple, but it still is beneficial for someone to explicitly point out just why all these concepts matter and how these concepts are applied in real life.

Let's look at how these concepts can be transferred to other modules:

  1. Computing every function that you write is essentially a system. If your function requires an input parameter and returns an output, those are your input and output signals. Same effect if a global variable is not inputted but is manipulated.
  2. Analog Electronics: the entire module is about operational amplifier, which is a system to amplify input signals to obtain desired output signals. Also can be configured as filters.
  3. Digital Electronics: a system comprising logic gates to control input signals to obtain desired output signals.
  4. Mathematics: every function is a system, every variable is a signal.
  5. Data Structure and Algorithms: a collections of systems and how to improve their performance.

While these analogies may not seem immediately obvious and certainly isn't formal representative of those topics, the author wishes to present this idea abstractly to help students new to this topic to gain awareness of where this course might lead you to.

As for all formal definitions and equations, please refer to your textbook and lecture notes.

Enjoy!
